MaxBettiNumbers :: maxBettiNumbers(..., Algorithm => ...)

maxBettiNumbers(..., Algorithm => ...)

Description

There are two algorithms that can be used to find the upper bounds given in maxBettiNumbers.

The “Simplified” algorithm simply finds the maximum of while ignoring the ideal structure of an ideal. In other words, it searches all possible numeric functions instead of just the Hilbert functions. This has two consequences. First, it is significantly faster because it allows for a simplification of the algorithm. Second, it does not always give the smallest upper bounds. However, there is one instance where it is guarenteed to give the smallest upper bounds: when no upper bound for the Hilbert function is specified.

The “Complete” algorithm does not make this simplification, and as a result, is slower but give the smallest upper bounds in every situation.

Ideally, we would like to use the “Simplified” algorithm when it gives the smallest upper bounds, and use the “Complete” algorithm otherwise. By default, the algorithm is selected that guarentees the smallest upper bounds. However, this can be overridden by passing the Algorithm option to maxBettiNumbers. The possible values are

  • "Automatic" - this is the default and chooses the algorithm to use based on the other inputs. Note: This will always give the smallest upper bounds as well as valid Hilbert functions.
  • "Simplified" - forces use of the “Simplified” algorithm. Note: if this option is passed, the values in “HilbertFunctions” may not be actual Hilbert functions.
  • "Complete" - forces use of the “Complete” algorithm. Note: This will always give the smallest upper bounds as well as valid Hilbert functions.

Further information

  • Default value: Automatic
  • Function: maxBettiNumbers -- Upper bounds for total Betti numbers in a family of saturated ideals.
  • Option name: Algorithm -- name for an optional argument

Caveat

If Algorithm=>"Simplified" is forced, this may not return valid Hilbert functions for some inputs.
